🚀 QUICK FIX PROCEDURE
═══════════════════════════════════════════════════════════════════

1. STOP SERVER
   Ctrl+C

2. CLEAR SESSION
   npm run fix-whatsapp

3. START FRESH
   npm run dev

4. SCAN QR CODE (only once!)
   Wait for: "✅ WhatsApp connected successfully!"
   Wait for: "Session saved. No need to scan QR code again"

5. VERIFY SESSION
   npm run test-session

6. TEST RESTART
   Ctrl+C
   npm run dev
   → Should connect WITHOUT QR code!

═══════════════════════════════════════════════════════════════════
🔍 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

ISSUE: QR code after every restart
───────────────────────────────────
CAUSE: Session files not saved
FIX:
  1. npm run test-session
  2. Check if baileys_auth/ has files
  3. If empty: npm run fix-whatsapp && npm run dev
  4. Scan QR and wait for "Session saved"

ISSUE: QR code appearing multiple times
────────────────────────────────────────
CAUSE: Connection issues or multiple init
FIX:
  - Just wait and scan the CURRENT QR code
  - Don't restart during scanning
  - Wait for "Session saved" message

ISSUE: Reconnection loops
─────────────────────────
CAUSE: Network instability
FIX:
  - Check internet connection
  - Wait 30 seconds between restarts
  - Service will auto-reconnect (up to 5 times)
  - Each retry uses longer delay (3s, 6s, 9s, 12s, 15s)

ISSUE: "QR code generated too quickly"
──────────────────────────────────────
CAUSE: Rate limiting protection active
FIX:
  - This is normal, not an error
  - Wait and scan current QR code
  - System prevents QR spam
